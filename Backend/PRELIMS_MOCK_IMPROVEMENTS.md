# Prelims Mock Improvements – Summary

## 1. Updated Question Schema

**PrelimsMock.questions[]** (and Test.questions[]):

- `questionId` (String, optional): Unique hash of question text for deduplication.
- `difficulty` (String): `"easy" | "moderate" | "hard"`.
- `question` (String): Question body; **HTML allowed** (e.g. `<table>...</table>`).
- `options`: `{ A, B, C, D }`.
- `correctAnswer`: `"A" | "B" | "C" | "D"`.
- `explanation` (Mixed): Either a **string** (legacy) or **option-wise object**:
  ```json
  {
    "A": "Why A is correct or incorrect (concept/source, elimination).",
    "B": "Why B is correct or incorrect.",
    "C": "Why C is correct or incorrect.",
    "D": "Why D is correct or incorrect."
  }
  ```

**PrelimsMock** (root):

- `title`: Custom test name (optional).
- `totalQuestions`: 100 (full-length) or 50 (sectional).
- `difficulty`: `"easy" | "moderate" | "hard"` (used for generation mix and filtering).
- `avoidPreviouslyUsed`: Boolean; when true, exclude snippets from past mocks are passed to the generator.

---

## 2. Test Generation Logic (Backend)

- **Deduplication**: After each full-mock run, `dedupeQuestions(questions)` is applied using `questionId` (hash of normalized question text). Only distinct questions are kept; then slice to `totalQuestions` (100 or 50).
- **Difficulty mix** (when “Moderate” is selected): 60% Moderate + 40% Hard (prompt-level). Easy/Hard have their own mixes.
- **Sectional (50 Q)**: For “Full Length GS Mix”, admin can choose 50 questions; generator runs 2 batches of 25 and dedupes.
- **GS + Current Affairs**: Full Mock Mix prompt explicitly includes Current Affairs (15–20 questions) with schemes, reports, international relations, environment updates.
- **Avoid previously used**: If `avoidPreviouslyUsed` is true, the controller fetches question snippets from other live/ended mocks and passes them to the generator so the model avoids repeating similar themes/wording.

---

## 3. No-Duplicate Selection (Mongo)

Questions are not selected from DB with a query; they are **generated by AI** and then **deduped in memory**:

```javascript
// In-memory dedupe (see testGenerationService.js)
function dedupeQuestions(questions) {
  const seen = new Set();
  return questions.filter((q) => {
    const id = q.questionId || hashQuestion(q.question);
    if (seen.has(id)) return false;
    seen.add(id);
    return true;
  });
}
// Then: const final = dedupeQuestions(all).slice(0, totalQuestions);
```

If you later store questions in a collection and **select** by criteria (e.g. by subject/difficulty), you can ensure distinct questions like this:

```javascript
// Example: get 100 distinct questions by subject/difficulty (hypothetical collection)
const pipeline = [
  { $match: { subject: "Polity", difficulty: "moderate" } },
  { $sample: { size: 200 } }, // oversample
  { $group: { _id: "$questionId", doc: { $first: "$$ROOT" } } },
  { $replaceRoot: { newRoot: "$doc" } },
  { $limit: 100 },
];
const questions = await Question.aggregate(pipeline);
```

## 4. Frontend: Table Support & Safe Rendering

- **Component**: `SafeQuestionHtml` (`Frontend/src/components/SafeQuestionHtml.tsx`).
- **When**: If `question.question` contains `"<table"`, the question body is rendered with `SafeQuestionHtml` (sanitized HTML).
- **Sanitization**: Only these tags are allowed: `table`, `thead`, `tbody`, `tr`, `td`, `th`, `p`, `br`, `strong`, `b`, `em`, `i`, `ul`, `ol`, `li`, `div`, `span`. All attributes are stripped. No script/event handlers.
- **Styling**: Table borders and cell padding are applied via Tailwind classes on the wrapper.

---

## 5. Frontend: Option-wise Explanation

- **TestResultPage**: If `question.explanation` is an object with `A`, `B`, `C`, or `D`, the “Show Explanation” section renders each option’s explanation separately (Option A: …, Option B: …, etc.). Otherwise it shows the legacy single string.

---

## 6. Admin Panel Additions

- **Test name**: Optional “Test name” input; stored in `title` and shown in the list.
- **Sectional (50 Q)**: For “Full Length GS Mix”, radio: “Full length (100 Q)” vs “Sectional (50 Q)”.
- **Difficulty**: Dropdown – Easy / Moderate (60% Moderate + 40% Hard) / Hard.
- **Avoid previously used questions**: Checkbox; when checked, snippets from other mocks are passed to the generator.
- **List filters**: Filter by **difficulty**, **subject** (substring on subject/title), **year** (for PYQ mocks: `yearFrom ≤ year ≤ yearTo`). “Apply” button refetches the list.

API: `POST /api/admin/prelims-mock` body can include `title`, `totalQuestions` (50 or 100), `difficulty`, `avoidPreviouslyUsed`.  
`GET /api/admin/prelims-mock?difficulty=moderate&subject=GS&year=2023` for filtered list.
